## 📖 ข้อมูลระบบ (System Overview)

Aetherium Genesis ได้วิวัฒนาการสู่สถาปัตยกรรม **AI Partner** เต็มรูปแบบ โดยเปลี่ยนมุมมองจาก LLM ที่เป็น "ตัวระบบทั้งหมด" สู่การเป็น "สมอง" ภายในร่างกายที่มีขอบเขตการทำงาน (Execution Vessels), มีระบบประสาทที่รวดเร็ว (AetherBus), และมีความจำที่ยั่งยืน (Memory Fabric)

### โครงสร้างหลัก (The 8 Layers):
1.  **Brain Layer (Cognitive Core)**: LLM สำหรับการวางแผนและวิเคราะห์ (AgioSage)
2.  **Execution Vessels (Body)**: ชั้นปฏิบัติการจริงที่มีสิทธิ์จำกัด (เช่น WorkspaceVessel)
3.  **Memory Fabric (Nervous System)**: ระบบประสาทและความจำ 6 ประเภท (Episodic, Semantic, Procedural, ฯลฯ)
4.  **Governance Core (Conscience)**: จริยธรรม กฎหมายภายใน และระบบตรวจสอบสิทธิ์ (ValidatorAgent)
5.  **Gems of Wisdom (Distilled Intelligence)**: หน่วยปัญญาตกผลึกจากประสบการณ์
6.  **RSI Engine (Evolution)**: วงจรพัฒนาตนเองภายใต้การกำกับ (Recursive Self-Improvement)
7.  **Identity Kernel (Persistent Self)**: แกนอัตลักษณ์ที่คงทนและไม่เปลี่ยนแปลงง่าย
8.  **Interaction Layer (Interface)**: ส่วนต่อประสานมนุษย์ (Neural Chat & Operational Dashboard)

## 🏛️ สถาปัตยกรรม AI Partner (Architecture)

ระบบทำงานประสานกันผ่านวงจร: **Think → Simulate → Authorize → Act → Reflect → Learn**

### 1. การไหลของข้อมูล (Operational Flow)
`Human Intent → Brain (Plan) → Governance (Audit/Tier) → Vessel (Simulate/Act) → Akashic Ledger (Record) → Reflector (Distill Gem)`

### 2. ระดับความเสี่ยง (Action Tiers)
- **Tier 0 (Read-only)**: อ่านข้อมูล, วิเคราะห์ (Auto-approve)
- **Tier 1 (Reversible Write)**: สร้าง Draft, แก้ไขไฟล์ (Policy-based approval)
- **Tier 2 (External Impact)**: ส่งอีเมล, โพสต์ข้อความ (Human-in-the-loop)
- **Tier 3 (Sensitive)**: ลบข้อมูลสำคัญ, ธุรกรรมการเงิน (Multi-layer confirmation)

## 🚀 การเริ่มต้นระบบ (System Awakening)

### 1. การเตรียมสภาพแวดล้อม
```bash
pip install -r requirements.txt
export PYTHONPATH=$PYTHONPATH:.
```

### 2. ปลุกระบบ (Awaken)
```bash
python awaken.py
```

เข้าใช้งานได้ที่:
- **Neural Chat Interface**: `http://localhost:8000`
- **Operational Dashboard (Governance & Gems)**: `http://localhost:8000/dashboard`
- **API Documentation**: `http://localhost:8000/docs`
